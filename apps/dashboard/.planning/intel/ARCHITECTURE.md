# Architecture

## What it does

`dashboard` is a stateless, SSO-aware aggregator at `dashboard.{{DOMAIN}}`. It fans out in
parallel to every registered data source's `/api/_internal/user-summary` endpoint (scoped by
Authentik UID), merges the results, and renders one card per source. It also shows a "Services"
grid populated from the auto-deploy pipeline's `ports.json` plus a static list.

## Tech stack

| Layer | Technology |
|-------|-----------|
| API | .NET 9 (`net9.0`), minimal APIs, no ORM, no DB |
| Frontend | React 19, Vite 6, TypeScript 5.7, Tailwind CSS 3 |
| API runtime | `mcr.microsoft.com/dotnet/aspnet:9.0` |
| Web runtime | `nginx:alpine` (serves built SPA) |

## Services in this repo

- **api** — .NET 9 minimal API; `network_mode: host`; exposes `/api/me`, `/api/aggregate`, `/api/services`, `/health`.
- **web** — nginx serving the React SPA; port-mapped `127.0.0.1:${WEB_PORT}:80`.

## External dependencies

- **Source apps** — called over host network (`127.0.0.1:<port>`) using shared `INTERNAL_SUMMARY_SECRET`; 1.5s per-source timeout.
- **Authentik** — Caddy forward auth injects `X-Authentik-*` headers; no Authentik SDK used.
- **`~/deploy/ports.json`** — mounted read-only at `/app/ports.json`; drives the services grid (pipeline service discovery).
- No message broker, no cache, no external DB.

## Data flow

1. Browser requests hit Caddy → Authentik forward auth → `X-Authentik-*` headers injected.
2. `GET /api/aggregate` → `AuthentikAuthHandler` extracts UID from `X-Authentik-Uid`.
3. API fans out in parallel to all enabled sources in `data-sources.json`; attaches `X-Internal-Secret`.
4. Each source returns a `SummaryPayload` JSON (or 204 = empty card, non-2xx = error badge).
5. Results assembled into `AggregateResponse`; returned to SPA as JSON.
6. React renders one `SourceCard` per result; polls every 60 s.

## Auth model

Authentik forward auth via Caddy `import authentik` → `X-Authentik-Username/Email/Uid/Groups` headers → `AuthentikAuthHandler` (`api/Common/Auth/AuthentikAuthHandler.cs`) → `[Authorize]` on endpoints. No JWT, no cookies generated by this app.
